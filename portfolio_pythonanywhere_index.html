Project 02 · 01/03 · 개요

ACC·AEBS 통합 종방향 제어 로직 설계

차량 속도·상대속도·상대거리를 기반으로 Cruise, Follow, AEB 모드를 전환하고, 선행 차량 추종과 돌발 상황 충돌 방지를 구현한 종방향 제어 프로젝트

연구 주제 — Simulink–CarMaker 기반 ACC·AEBS 통합 제어기 설계

운전자가 설정한 속도를 유지하면서 선행 차량이 감지되면 안전거리를 확보하고, 돌발 장애물 발생 시 충돌 전 정지시키는 통합 제어 로직을 설계하였다. Driver / Cruise / Follow / AEB 4개 모드로 구성하고, 차량 속도·상대거리·상대속도·객체 감지 결과에 따라 자동 전환되도록 구현하였다.

프로젝트 필요성

ACC는 설정 속도 유지뿐 아니라 주변 차량 움직임에 따라 목표 속도와 안전거리를 함께 바꿔야 하는 시스템이다. 실제 주행에서는 다음 상황이 연속적으로 발생한다.

  • 선행 차량이 없는 구간에서 설정 속도 유지
  • 느린 선행 차량 등장 시 감속 및 추종
  • 선행 차량 이탈 시 설정 속도로 재가속
  • 정지 차량·돌발 장애물 등장 시 긴급 제동
  • 각 제어 모드 간 출력 중첩·간섭 방지

연구 방법

Cruise Mode는 설정속도-차속 오차 기반 PI 제어(Kp=1000, Ki=0.0001)로, Follow Mode는 속도·거리 오차에 가중치(0.78 / 0.35)를 부여한 복합 오차를 PID 제어(Kp=1400, Ki=1e-10, Kd=50)로 처리했다. Time Gap(2.5~3.0초) 기반으로 목표 거리를 계산하고, AEB는 TTC 기반 3단계 순차 감속으로 별도 구성했으며, Stateflow에서 모드별 0/1 스위치 신호로 출력 중첩을 방지하였다.

Project 03 · 01/03 · 개요

차량 소음 분석 및 구조물 Modal Test 기반 NVH 특성 평가

실차 주행 소음을 주파수 영역에서 분석하고, 임팩트 해머 기반 모달 테스트를 통해 구조물의 고유진동수와 모드 형상을 도출한 NVH 분석 프로젝트

연구 주제 — MATLAB 기반 차량 소음 분석 및 1/3 Octave Band 평가

실차에서 직접 측정한 소음 데이터를 MATLAB으로 보정하고 FFT, 1/3 Octave Band, A-Weighting 분석을 수행하여 주요 소음 주파수와 특성을 정량적으로 확인하였다. 분석 결과를 바탕으로 흡·차음재, 엔진 마운트, ANC 등 적용 가능한 소음 저감 방안을 제안하였다.

프로젝트 필요성

실차 소음의 원인을 정량적으로 파악하고, 구조물의 고유 진동 특성을 실험적으로 검증할 필요가 있었다.

  • 실차 주행 소음의 주파수 대역별 특성 파악 필요
  • 인간 청각 특성을 반영한 체감 소음 평가 필요
  • 구조물의 고유진동수·모드 형상을 실험적으로 검증할 필요
  • 저주파 공진 위험 구간의 사전 파악 필요

연구 방법

스마트폰으로 측정한 소음 신호의 SPL을 보정 상수(약 2.4774)로 보정하고, FFT와 1/3 Octave Band(직접 구현)·A-Weighting으로 주파수 특성을 분석하였다. 별도로 철판 구조물에 임팩트 해머 가진 후 가속도계로 21개 포인트를 측정, FRF·Phase·Coherence 조건을 함께 확인해 고유진동수를 판별하고 spaps/csaps 보간으로 Mode Shape를 시각화하였다.

문제 01

내장 Octave Filter 결과 불일치

MATLAB 내장함수 적용 결과가 기대한 주파수 밴드와 다르게 나타났다.

  • 내장함수 대신 1/3 Octave Band 정의·중심주파수 공식 재검토
  • 밴드별 주파수 범위를 직접 계산해 필터링 코드 작성
  • 중심주파수별 SPL 그래프를 직접 생성
문제 02

앱 측정값과 MATLAB SPL 차이

스마트폰 앱은 72dB, MATLAB 원신호 계산은 약 64.12dB로 약 8dB 차이가 발생했다.

  • 측정 앱과 원신호의 레벨 차이 확인
  • 보정 상수(약 2.4774) 산출 및 원신호 적용
  • 보정 후 72dB 일치 여부 재확인
문제 03

측정점 21개 중 일부 피크 누락

모달 테스트에서 예상한 21개 피크 대신 19개만 확인되었다.

  • 가속도계 위치, 타격 편차, Double Hit, 케이블 노이즈 등 원인 분석
  • FRF 피크만이 아닌 Phase·Coherence 동시 확인으로 대응
  • 이상 차수는 별도 표시 후 전체 부호 변화 추세 중심 해석
문제 04

고차 모드 형상 일부 불일치

모드 차수가 증가할수록 Node·부호 변화가 늘어나야 하지만, 일부 고차 모드에서 예상 형상과 어긋났다.

  • 오류 차수를 별도 표시
  • 그래프의 부호·개형 변화를 함께 검토
  • 실험 장비·조건의 한계를 명시하고 확실한 결과와 구분해 제시
Project 03 · 03/03 · 확보한 역량

프로젝트를 통해 확보한 역량

01

신호 처리 역량

시간 영역의 원시 데이터를 주파수 영역으로 변환하고, 필요한 정보를 추출하는 전체 흐름을 경험하였다.

02

MATLAB 데이터 분석 역량

내장함수가 원하는 결과를 제공하지 않을 때, 분석 원리를 기반으로 직접 코드를 구현하였다.

03

실험 데이터 해석 역량

FRF 피크만을 고유진동수로 단정하지 않고, Phase와 Coherence를 함께 검토해 판별하였다.

04

실험 오차 분석 역량

센서 위치, 케이블 접촉, 타격 편차, Double Hit 등 실험 오차의 원인을 구체적으로 분석하였다.

05

NVH 개선안 도출 역량

단순 분석에서 끝나지 않고 공기 전달음, 구조 전달음, 능동 소음 제어(ANC) 관점의 실질적 개선안을 제시하였다.
